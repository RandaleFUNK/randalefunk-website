# AGENTS.md

Projektregeln fuer Codex-Arbeit an RandaleFUNK.

## Standard-Workflow fuer Website-Aenderungen

1. Aenderungen an der Website werden zunaechst nur im GitHub-Repository vorgenommen.
2. Codex darf Aenderungen committen und nach GitHub uebertragen.
3. Nach jeder Aenderung nennt Codex:
   - welche Dateien geaendert wurden,
   - was geaendert wurde,
   - den Commit oder Pull Request.
4. Es erfolgt keine automatische Veroeffentlichung ohne ausdrueckliche Freigabe.
5. Erst wenn der Nutzer ausdruecklich "Freigabe", "Veroeffentlichen" oder eine gleichwertige Anweisung erteilt, darf der Deploy-Prozess gestartet werden.
6. Die Veroeffentlichung erfolgt ueber die bestehende GitHub-Action auf den United-Domains-Webspace von randalefunk.de.
7. Wenn keine Freigabe vorliegt, bleiben die Aenderungen ausschliesslich im GitHub-Repository.

Grundsatz: GitHub ist die Arbeits- und Pruefversion. randalefunk.de ist die veroeffentlichte Live-Version. Zwischen beiden erfolgt die Veroeffentlichung nur nach ausdruecklicher Freigabe.
